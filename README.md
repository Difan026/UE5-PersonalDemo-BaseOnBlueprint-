# 第三人称闯关游戏 - UE5蓝图项目

![UE5](https://img.shields.io/badge/Unreal%20Engine-5.3-blue) ![Blueprint](https://img.shields.io/badge/Blueprint-100%25-brightgreen)

## 📖 项目简介

一款基于 **Unreal Engine 5 蓝图** 独立开发的第三人称闯关游戏Demo。玩家控制角色在关卡中探索，收集能量球和金币，与宝箱、机关、电梯等物体交互，使用商店购买生命药瓶，管理生命值并最终达成胜利条件。

本项目完全使用UE(5.7)蓝图实现，通过蓝图接口、事件分发器、GameInstanc、函数封装等手段保证系统的**可扩展性**、**可维护性**和**性能**。

---

## 🎮 核心功能

| 模块 | 描述 |
|------|------|
| **角色系统** | 第三人称移动、生命值管理、受伤/死亡反馈、重生机制、资源管理 |
| **交互系统** | 事件重叠检测触发交互，支持能量球、宝箱、机关、电梯等多种物体 |
| **资源与背包** | HUD实时显示能量球/金币数量，数据跨关卡持久化 |
| **商店系统** | 购买生命药瓶，金币与角色资源实时挂钩，购买反馈完善 |
| **药品使用** | 背包中使用药瓶，条件判断（生命未满且有药瓶），UI同步更新 |
| **关卡流程** | 基于能量球收集数的检查点判定，控制电梯解锁；胜利/失败/暂停界面 |
| **音效反馈** | 角色受伤、物体拾取、宝箱开启、电梯运行等全流程音效 |
| **碰撞优化** | 摆锤机关采用 `OnHit + ApplyDamage` 避免死亡流程中断 |

## 🛠️ 技术实现亮点

###  蓝图接口（Interfaces）
所有可交互物体（能量球、宝箱、机关、电梯）实现统一的 `Interact` 接口，玩家射线检测后直接调用接口函数，新增交互类型无需修改玩家逻辑。

###  事件分发器（Event Dispatchers）
UI与数据解耦：资源数量变化时通过事件分发器通知HUD刷新，避免每帧轮询或强耦合。

###  GameInstance 跨关卡持久化
能量球、金币数量存储在自定义 `GameInstance` 中，关卡切换不丢失。

###  音效与对象生命周期解耦
使用 `Play Sound at Location` 节点在碰撞或交互事件中直接播放世界位置音效，避免挂载音效组件因物体销毁而无法播放的问题。该方案统一应用于：拾取物品、开启宝箱、启动电梯、角色受伤等场景。

###  碰撞伤害优化
摆锤机关等伤害源使用 **`OnHit` 命中事件** 替代 **`OnOverlap` 重叠事件**，调用 `ApplyDamage` 并传入角色生命上限，由角色自身受伤系统统一处理死亡逻辑。解决重叠事件反复触发导致死亡流程中断的问题，同时省去 `Cast To` 类型转换，提升性能。

###  代码组织
- 重复逻辑封装为**函数**（如受伤处理、UI刷新）
- 常用节点组合制作**宏**（如条件判断+日志输出）
- 关键系统采用**组件化**设计（如资源管理组件、交互检测组件）

## 📁 项目结构（核心蓝图）

```
Content/
├── Blueprints/
│   ├── Character/        # 角色蓝图（BP_ChallengeCharacter）
│   ├── Interactables/    # 交互物体（BP_Orbs, BP_Sphere, BP_EasyShop...）
│   ├── GameInstance/     # 自定义GameInstance（存储资源数据）
│   └── Input/            # 输入检测
├── UI/                   # 控件蓝图（HUD, 背包, 商店, 暂停菜单...）
├── Maps/                 # 关卡地图
└── Audio/                # 音效资源
```
